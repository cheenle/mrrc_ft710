Full Control. Any Device.

Browser-based remote control for the Yaesu FT-710 HF/50MHz transceiver.
Real-time waterfall spectrum, bidirectional Opus audio, iPhone-optimized mobile UI — complete operation from a single phone.

GitHub Repository Quickstart

Core Features

🎛️

Full Rig Control

Frequency, mode, band, VFO, filters, preamp, attenuator, NR/NB/AN — all CAT commands over WebSocket in real time, <20ms latency at 38400 baud.

📊

Real-Time Waterfall

850-point FFT spectrum + 120-line history waterfall, ~30fps. Dual-mode: FT4222 SPI real spectrum + S-meter synthetic fallback.

🎤

Bidirectional Opus Audio

RX: Radio → Opus 64kbps → browser AudioWorklet. TX: Phone mic → Opus encode → radio TX. Jitter buffer + graceful PTT release preserves word endings.

📱

Mobile-First UI

Optimized for iPhone Safari: safe-area, PWA manifest, Service Worker offline cache. Touch-and-hold PTT — release to stop TX instantly.

🔐

Safety First

TLS/HTTPS encryption, token auth, PTT watchdog (500ms), dead-man switch, client disconnect auto-forces RX.

⚡

Dead Simple Deployment

Single Python process: FastAPI + Uvicorn async server. macOS / Linux / Raspberry Pi. pip install and run.

System Architecture

  📱 Browser (iPhone / Desktop / Tablet)
    │  ↕ HTTPS + WebSocket × 4
  🖥️  MRRC FT-710 Server (Python FastAPI + Uvicorn)
    │
    ├── 📡 Serial CAT (USB Enhanced COM Port, 38400 baud)
    ├── 🔬 FT4222 SPI → scope_pipe subprocess → real FFT spectrum
    ├── 🎙️  PyAudio USB Audio (RX capture / TX playback)
    └── 🗜️  libopus (ctypes): RxOpusEncoder + TxOpusDecoder
  📻  Yaesu FT-710 HF/50MHz Transceiver

No wfview / Hamlib / any external radio middleware. CAT commands go direct over serial. Spectrum reads from the FTDI FT4222 SPI chip. Audio via the radio's built-in USB sound card.

Quickstart

# 1. Install dependencies
cd FT710
pip install -r requirements.txt

# 2. Start (macOS)
FT710_SERIAL_PORT=/dev/cu.usbserial-0121DB3A0 python server.py

# 2. Start (Linux / Raspberry Pi)
FT710_SERIAL_PORT=/dev/ttyUSB0 python server.py

PTT Safety Architecture

Transmit safety is the highest design priority. MRRC FT-710 implements multiple defense layers:

  1. Touch-and-hold → instant release. PTT uses touch-and-hold interaction. Mouse leave / touch cancel releases immediately.
  2. Server-side TX status poll. 0.5s TX; query detects TX→RX transitions and zeroes meters immediately.
  3. PTT watchdog. Client-side 500ms timer verifies RX state; retries TX0; up to 3× on anomaly.
  4. Server-side forced RX. When the last client disconnects during TX, server auto-sends TX0;.
  5. Browser unload beacon. beforeunload + pagehide events force TX0; on tab close.
  6. Graceful TX audio drain. On PTT off, queued audio is written to DAC (50ms) before RF drops — word endings preserved.
